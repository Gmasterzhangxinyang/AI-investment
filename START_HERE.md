# AI 投研工作台启动说明

这是 macOS Apple 芯片离线交付版。完整交付包已包含 Python 和全部运行依赖，用户不需要进终端，也不需要提前安装 Python。

## 1. 放入 Excel

把三个 Wind Excel 放到：

```text
data/wind/current/
```

文件名保持为：

```text
01_ETF清单和日频公式.xlsx
02_TL日频公式.xlsx
03_可转债数据.xlsx
```

## 2. 第一次使用先自检

macOS 下第一次使用，先双击：

```text
初始化环境.command
```

完整离线包会直接检查内置运行环境，不会联网安装，也不会修改电脑的系统 Python。自检只需要做一次。

如果项目中没有 `runtime/` 目录，说明拿到的不是完整离线交付包；此时脚本才会退回开发版的 Python 安装流程。

## 3. 双击启动

macOS 下双击：

```text
启动AI投研.command
```

脚本会自动：

- 创建 `data/wind/current/`、`outputs/latest/`、`logs/`
- 检查三个 Excel 是否存在
- 优先使用交付包内置 Python 和运行依赖
- 启动本地服务 `http://127.0.0.1:8766/frontend/`
- 自动打开浏览器

## 4. 刷新数据

进入页面后点击：

```text
一键刷新
```

系统会读取三个 Excel，生成 dashboard、日报、SQLite 数据和前端页面。

刷新时旧结果会继续保留。新结果全部校验完成后才一次性切换；如果刷新失败，页面会显示具体失败环节和处理建议，不会用半成品覆盖上一次结果。

## 注意

- 关闭启动窗口会停止本地服务。
- 如果缺少某个 Excel，系统仍会启动；对应模块会降级显示，其他模块继续可用。
- 如果三个核心 Excel 都不存在，前端刷新会提示源文件缺失。
- 投研问答中的查数、单标的和多 ETF 当前状态由本地规则快速回答；开启 AI 后，普通解释使用快速模型，跨资产或复杂问题才进入只能调用只读白名单工具的 ReAct 主 Agent。
- 如果端口 8766 被占用，可以用终端临时指定端口：

```bash
AI_RESEARCH_PORT=8770 ./启动AI投研.command
```
