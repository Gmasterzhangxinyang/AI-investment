# AI 投研系统产品使用说明

| 项目 | 当前内容 |
| --- | --- |
| 版本 | V2.0（策略插件与 Research Agent 版） |
| 场景 | 日度 ETF、30 年国债期货 TL、可转债投研辅助 |
| 数据 | 本地 Wind Excel |
| 输出 | 看板、日报、Excel、PDF、SQLite、投研问答 |

## 1. 系统定位

系统每天收盘后读取本地 Wind 数据，完成数据校验、指标计算、策略筛选、风险识别、报告生成和数据库入库。交易状态、评分、排名和风险标记全部由确定性代码生成；AI 只读取受控证据并解释，不能修改结果。

当前核心：

- ETF：默认 `trend_pullback_v2`，同时看中期趋势与短期入场。
- TL：日频 MACD/KDJ 规则，周线不利为硬否决，ETF 份额变化仅作辅助。
- 可转债：`legacy_v1` 基础排名，加 `dynamic_v2` 四项短期辅助。
- 问答：规则快速路径、AI 快速解释和受控 ReAct 主 Agent 三档。
- 刷新：先 staging，成功后原子发布；失败保留上一版完整结果。

## 2. 每日使用

1. 在三个 Wind Excel 中刷新到最新交易日。
2. 将文件放入 `data/wind/current/`，保持标准文件名。
3. 双击 `启动AI投研.command`。
4. 在网页点击“一键刷新”。
5. 查看数据状态确认三类数据读取成功。
6. 再查看 ETF、TL、可转债和每日报告。
7. 需要追问时进入投研问答；只有需要自然语言解释时才开启 AI。

修改策略或参数后必须重新刷新。保存配置本身不会改写已经生成的日报。

## 3. 页面

| 页面 | 用途 |
| --- | --- |
| 投研问答 | 查数、解释状态、比较标的、查看数据范围；复杂问题可由主 Agent 选工具。 |
| 每日报告 | ETF、TL、可转债和风险摘要。 |
| ETF | 中期趋势、短期入场、建仓候选、关注池、平仓提示和全量排名。 |
| TL | 当前状态、日/周指标和 30 年国债 ETF 份额辅助。 |
| 可转债 | 合格候选、弱观察、风险观察、排除列表和动态辅助。 |
| 标的档案 | 单标的当前指标及有限历史。 |
| 数据状态 | 刷新结果、三类数据覆盖、失败阶段和处理建议。 |
| 策略参数 | 选择 ETF 插件、启停可转债辅助、调整阈值与模型。 |
| 参数说明 | 当前参数含义和调高/调低影响。 |

## 4. ETF

### 4.1 当前插件

`etf.active_strategy=trend_pullback_v2`。原策略 `legacy_v1` 仍保留，可切换并参与历史对照，但不是当前默认。

### 4.2 中期趋势

出现 MA20 继续向下或周 MACD 绿柱扩大等硬否决时，系统标记“不参与”。没有硬否决后，价格与 MA5 站上 MA20、MA20 走平或向上、周/日 MACD 共同确认，才进入“趋势已确认”。

### 4.3 密切观察

```text
MA5 已在 MA10 上方
且日 MACD 绿柱缩短或转红
=> 密切观察
```

系统同时提示周 MACD 是否绿柱缩短/红柱加长，以及 MA20 是否走平。密切观察不是趋势确认，也不是建仓。

### 4.4 短期入场

中期趋势确认后，系统再等待：

- 收盘突破趋势确认日高点，日 MACD 保持红柱，且价格高于 MA5 不超过默认 3%；或
- 缩量回踩 MA5、MA10 或突破位后重新站稳。

长期下跌后的巨量长阳、短期涨幅/实体/量能/MA5 偏离同时过热时，系统提示“过热不追”并进入冷却。

### 4.5 持仓退出

只有当前持有的 ETF 才检查：

- 收盘跌破 MA10 且量能倍数达到 1.2；或
- 收盘跌破 MA5 且量能倍数达到 1.2。

### 4.6 关键参数

| 参数 | 当前值 |
| --- | ---: |
| MA20 斜率回看 | 5 日 |
| MA20 走平容忍度 | 0.3% |
| 过热单日涨幅 | 4% |
| 过热阳线实体占比 | 60% |
| 过热量能倍数 | 1.8 |
| 过热 MA5 偏离 | 3% |
| 冷却期 | 3 日 |
| 突破/回踩设置最长保留 | 10 日 |
| MA10/MA5 平仓量能 | 1.2 / 1.2 |

评分权重仍为趋势 35%、MACD 25%、量能 25%、份额变化 15%，只用于排序，不替代状态条件。

## 5. TL

TL 输出三类状态：不做交易、关注交易、模型触发建仓候选。

- 周线 MACD 红柱缩短、绿柱加长或红转绿属于不利。
- 周线近 2 周最低 J 需低于 20，且当前 J 回升。
- 日线近 3 日最低 J 需低于 5，且当前 J 回升。
- 周线不利硬否决开启时，日线不能覆盖周线。

30 年国债 ETF 份额变化按收盘后数据使用：

| 绝对变化 | 当前解释 |
| --- | --- |
| 0.03 亿份 | 轻度变化 |
| 0.05 亿份 | 较大变化 |
| 0.07 亿份 | 极值参考 |

份额变化和滚动方向只提示潜在拐点，不改变 TL 主状态。

## 6. 可转债

### 6.1 基础决策层

`legacy_v1` 负责：

1. 硬排除价格、强赎、ST、低评级、异常 YTM、高溢价和过小规模。
2. 按基本面 25%、溢价 20%、YTM 15%、信用 15%、期限 10%、强赎 10%、规模 5% 计算基础分。
3. 决定基础等级、资格、动作和最终排名。
4. 从合格候选中做行业分散，不用弱观察或风险观察凑满 Top10。

关键风控：价格 100 至 140 元区间、高 YTM 15%、严重负 YTM -5%、高溢价扣分线 35%、硬排除 50%、最小存续规模 0.5、A/A- 及以下评级硬排除、不强赎公告有效期 180 日。

### 6.2 dynamic_v2 四项辅助

当前默认启用，读取：

1. 正股当日涨跌。
2. 转债当日涨跌。
3. 前一日转股溢价率。
4. 当日转股溢价率变化。

输出关注补涨、谨慎追涨、联动走弱、正常联动或数据不足。

重要：动态辅助不改变硬排除、基础分、资格、动作和最终排名。代码保持 `score = base_score`，`dynamic_score` 只作为辅助字段。配置里的 0.8/0.2 当前是插件元数据，没有合成为最终排名分。

## 7. 投研问答

### 7.1 三档运行

| 路径 | 问题 | 处理 |
| --- | --- | --- |
| 规则快速 | 收盘、评分、参数、当前状态、名单、多 ETF 状态 | 后端工具取数后直接回答，不调用 LLM。 |
| AI 快速解释 | 对象明确的“为什么、怎么理解” | 固定证据包交给快速模型解释。 |
| ReAct 主 Agent | 跨资产、复杂排序或对象不明确 | 主 Agent 自主选择白名单工具，检查证据并反思后回答。 |

### 7.2 数据范围

- ETF 当前排序：已入库 ETF 横截面，单次最多显示 10 只。
- ETF 单标的：最近 30 个交易日。
- ETF 多标的：一次 2 至 10 只，逐只读取当前状态。
- ETF 双策略：单只最多使用 400 个交易日计算。
- TL：最近 30 个交易日；页面显示 8 日不限制问答。
- 可转债：问答最多读取前 30 只；页面 Top10 不限制问答。
- 浏览器保留最近 8 轮，本次模型提示最多使用最近 4 轮。

### 7.3 权限

主 Agent 只能调用注册的只读工具。它不能直接执行 SQL、读取任意本地文件、访问互联网新闻、刷新数据、写库、改配置、修改信号或下单。每份证据先经过准确性审查，最终回答再经过输出保护。

模型缺 Key、超时或调用失败时，系统回退到确定性回答，不影响策略和日报。

## 8. 数据与刷新

标准文件：

```text
data/wind/current/01_ETF清单和日频公式.xlsx
data/wind/current/02_TL日频公式.xlsx
data/wind/current/03_可转债数据.xlsx
```

刷新过程：

```text
读取与计算
  -> outputs/.staging/<run_id>
  -> 报告与质检
  -> SQLite 入库
  -> 原子发布 dashboard.json
```

失败时页面说明失败阶段、影响和处理建议，上一版成功结果继续可用。页面刷新不会取消后台任务。

## 9. 历史诊断

ETF v2 统计状态后 1/3/5/10/20 日收益分布、最大有利/不利波动和假反转，只用于检查历史表现。当前没有完整组合资金曲线、仓位管理、滑点建模、年化收益、Sharpe 或正式样本外回测。

## 10. 当前边界

已实现：本地三类 Wind 数据、ETF v2、TL 日频与份额辅助、可转债动态辅助、原子刷新、SQLite、日报/PDF、规则问答、AI 快速解释、只读 ReAct Agent、证据审查和输出保护。

未接入：互联网新闻、实时行情、交易账户、自动下单、组合级正式回测。
