# Data Quality Rules

数据质量层的目标不是阻止每天出报告，而是让客户清楚知道哪些模块可用、哪些模块不可用、为什么不可用。

## 通用规则

系统会检查：

- 源文件是否存在。
- Excel 是否可解析。
- 必要字段是否齐全。
- 日期和代码是否重复。
- 开盘价、收盘价是否非正。
- 成交量是否为空或为 0。
- 有效历史长度是否足够。
- ETF 和 TL 最新日期是否一致。

## 缺文件处理

缺少某个 Excel 时：

- 系统不崩溃。
- CLI 和前端刷新都继续调用日更流程。
- 对应模块输出 `unavailable` 或 `data_unavailable`。
- `data_quality.errors` 写入原因。
- 其他模块继续正常计算。

前端只在 ETF、TL、可转债等全部配置的核心源 Excel 都不存在时，才会在启动日更前返回明确错误。

## 非交易行和零成交量

Wind Excel 中出现以下情况时，该行视为非有效交易行：

- `open <= 0`
- `close <= 0`
- `volume <= 0`
- 开盘价、最高价、最低价、收盘价或成交量缺失

- 不进入指标计算。
- 不进入 ETF 前 60 日均量。
- 不进入 TL MACD/KDJ。
- 计入 `data_quality` 的无效交易行过滤数。

## 历史长度

ETF 和 TL 低于 60 个有效交易日时标记 `ERROR`，低于 120 个有效交易日时标记 `WARN`。

- ETF 有效历史不足 60 日：输出 `data_unavailable`，不允许输出 `buy_candidate`。
- TL 有效历史不足 60 日：输出 `unavailable`，不允许输出 `entry_candidate`。
- 历史不足时可以生成流程报告，但不能输出高置信建仓结论。

## 数据质量如何影响 confidence

- `OK`: 可按策略规则给出 `high/medium`。
- `WARN`: 至少降一级，通常不高于 `medium`。
- `ERROR`: 交易类信号强制降为 `low`，且不能输出高置信建仓候选。

## 可转债

可转债会检查：

- 价格。
- 剩余期限。
- 转股溢价率。
- 到期收益率。
- 强赎触发价、赎回公告、不强赎公告。
- 债项评级。
- 申万行业。
- 存续规模。
- 扣非净利润增长与增长加速。

强赎字段缺失时，系统不会自行补全公告结论；该标的 `confidence` 降为 `low` 并提示需人工复核。触发强赎价但没有有效不强赎公告时，默认硬排除，不进入普通 Top候选。

可转债缺失时不会影响 ETF/TL 日报生成，但 `convertible_bond.status` 会显示 `unavailable`。
