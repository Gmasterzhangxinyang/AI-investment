# Data Quality Rules

数据质量层的目标不是阻止每天出报告，而是让客户清楚知道哪些模块可用、哪些模块不可用、为什么不可用。

## 通用规则

系统会检查：

- 源文件是否存在。
- Excel 是否可解析。
- 必要字段是否齐全。
- 日期和代码是否重复。
- 开盘价、收盘价是否非正。
- 成交量是否为空或为 0。
- 有效历史长度是否足够。
- ETF 和 TL 最新日期是否一致。

## 缺文件处理

缺少某个 Excel 时：

- 系统不崩溃。
- 对应模块输出 `unavailable` 或 `data_unavailable`。
- `data_quality.errors` 写入原因。
- 其他模块继续正常计算。

## 非交易行和零成交量

Wind Excel 中成交量为 0 或价格为 0 的行视为非有效交易行：

- 不进入指标计算。
- 不进入 ETF 前 60 日均量。
- 不进入 TL MACD/KDJ。
- 质检层保留可见提示。

## 历史长度

ETF 和 TL 低于 60 个有效交易日时标记 `ERROR`，低于 120 个有效交易日时标记 `WARN`。历史不足时可以生成流程报告，但不能输出高置信建仓结论。

## 可转债

可转债会检查：

- 价格。
- 剩余期限。
- 转股溢价率。
- 到期收益率。
- 强赎触发价、赎回公告、不强赎公告。
- 债项评级。
- 申万行业。
- 存续规模。
- 扣非净利润增长与增长加速。

可转债缺失时不会影响 ETF/TL 日报生成，但 `convertible_bond.status` 会显示 `unavailable`。
