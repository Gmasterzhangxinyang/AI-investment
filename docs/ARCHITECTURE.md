# Architecture

## Design principle

The system uses a production-shaped Agent + Skill + Tool structure:

```text
Orchestrator
  -> Agents
    -> Skills
      -> Tools
```

Signals are deterministic. LLM output may explain signals but must not change them.

## Atomic refresh and publish

Daily refresh is a two-phase operation:

```text
read/compute
  -> outputs/.staging/<run_id>
  -> QA audit against staged artifacts
  -> SQLite ingest against staged artifacts
  -> publish report / indicators / audit
  -> publish dashboard.json last (atomic replace)
```

The public dashboard is never replaced by a partially written run. Any failure leaves the previous successful dashboard available. Refresh jobs store structured progress and a user-facing diagnosis for source ingestion, indicators, ETF, TL, convertible bonds, historical diagnostics, reporting, audit, database ingest, and runtime failures.

## Runtime

| File | Responsibility |
|---|---|
| `runtime/context.py` | Shared blackboard for one workflow run |
| `runtime/result.py` | Agent and workflow result contracts |
| `runtime/agent.py` | Base Agent execution wrapper, AgentSpec contracts, and SkillBackedAgent |
| `runtime/orchestrator.py` | Sequential workflow orchestrator |
| `runtime/skill_registry.py` | Discovers `SKILL.md` and runs handlers |
| `runtime/artifact_store.py` | Saves JSON/CSV artifacts |
| `runtime/audit_logger.py` | Writes audit JSONL |

## Agents

Agents are governance units. Each Agent declares its role, objective, required inputs, expected outputs, quality gates, and decision policy. The runtime validates required inputs before execution and verifies expected outputs after execution. Full contracts are documented in `docs/AGENTS.md`.

| Agent | Purpose |
|---|---|
| ConfigAgent | Load strategy, universe, model and report/runtime config |
| SourceArchiveAgent | Archive configured source files and write a source manifest |
| DataAgent | Read ETF, TL, and convertible-bond Wind Excel data |
| QAAgent | Validate data quality |
| IndicatorAgent | Calculate technical indicators |
| PortfolioAgent | Load configured position state |
| ETFAgent | Generate ETF signals |
| TLAgent | Generate TL timing state |
| ConvertibleBondAgent | Rank convertible bonds after risk filters and industry diversification |
| BacktestAgent | Produce lightweight historical diagnostics and trade evidence |
| RiskAgent | Summarize portfolio risk |
| AIResearchCommitteeAgent | Run optional constrained evidence review or deterministic fallback commentary |
| ExplanationAgent | Generate report commentary from existing evidence |
| ReportAgent | Write Excel report and frontend JSON |

## Skills

Every Skill has:

```text
SKILL.md     # metadata and usage contract
handler.py   # deterministic execution
```

This mirrors the official skill pattern while keeping production execution under code control.

## Frontend

The frontend is a static local research dashboard. It reads:

```text
outputs/latest/dashboard.json
```

It does not calculate strategy logic. It displays, filters, and explains outputs generated by the backend. 投研问答默认使用规则证据包回答；用户点击 `AI 智能问答` 并确认后，才调用大模型进行自然语言增强解释。

## Chat runtime

```text
Question
  -> deterministic Router
  -> rule fast lane | focused AI lane | bounded ReAct deep lane
  -> evidence/accuracy checks
  -> output guardrail
  -> trace persistence
```

The chat Agent never receives raw SQL or unrestricted filesystem/network tools. Tool permissions and row limits are code-level contracts in `chat/tool_registry.py` and `chat/access_policy.py`. See `docs/CHAT_AGENT_RUNTIME.md`.
