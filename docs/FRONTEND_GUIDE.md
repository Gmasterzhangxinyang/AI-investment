# Frontend Guide

The frontend is a local research workstation served by `serve.py`. It displays backend-generated evidence and does not calculate strategy logic in the browser.

## Start

Start a local server from the project root:

```bash
python serve.py --port 8766
```

Open:

```text
http://127.0.0.1:8766/frontend/
```

`serve.py` defaults to port `8766`. Use `--port` only when you need a custom local port.

## Data Source

The frontend reads:

```text
outputs/latest/dashboard.json
```

It also uses local API endpoints for refresh jobs, PDF export, chat, asset details, database status, and strategy parameter reads/writes.

## Actions

- `Refresh Data`: calls `POST /api/refresh`, runs the daily workflow, then reloads `outputs/latest/dashboard.json`.
- `Export PDF`: calls `POST /api/export-pdf` and opens a Chinese-safe PDF generated by the local backend.
- `Open Report`: links to the generated Excel workbook.
- `Save Strategy Params`: writes `configs/strategy_params.json`; users must refresh data before new parameters affect reports.

## Missing Excel Behavior

Frontend refresh follows the same degradation rule as the CLI:

- If ETF, TL, or convertible-bond Excel is missing individually, refresh still starts.
- The ingestion layer creates empty module frames and records data quality warnings.
- The missing module displays `unavailable` or `data_unavailable`; other modules continue.
- Only when all configured core source Excel files are missing does refresh return an immediate error.
