# Frontend Guide

The frontend is a static, international-style research workstation.

## Pages

- Overview
- ETF Signals
- ETF Watchlist
- TL Timing
- Risk
- Agent Audit

## Visual direction

- high information density
- restrained neutral palette
- clear risk/status colors
- table-first investment workflow
- no marketing landing page
- no decorative background effects

## Data source

```text
outputs/latest/dashboard.json
```

Start a local server from the project root:

```bash
python serve.py
```

Open:

```text
http://127.0.0.1:8765/frontend/
```

## Actions

- Refresh Data: calls `POST /api/refresh`, runs the daily workflow, then reloads `outputs/latest/dashboard.json`
- Export PDF: calls `POST /api/export-pdf` and opens a Chinese-safe PDF generated by the local backend
- Open Report: links to the generated Excel workbook

The ETF Watchlist section is interactive. Selecting one ETF shows:

- watch type
- missing condition
- suggested action
- MA5/MA10 and MA5/MA20 status
- recent 8 trading days of close, MA5, MA10, MA20, volume ratio, MACD histogram, and KDJ J

`Refresh Data` requires `serve.py`; it will not work from a plain static `python -m http.server`.
