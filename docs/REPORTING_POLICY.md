# Reporting Policy

本系统是规则模型生成的投研辅助工具，不构成投资建议或收益承诺。所有用户可见输出必须使用保守、可审计、可复核的话术。

## 必须使用的表达

- `模型触发建仓候选`
- `模型触发平仓提示`
- `进入观察池`
- `需人工复核`
- `历史诊断显示`
- `不构成投资建议`
- `不做交易`
- `关注交易`
- `数据不足，无法判断`

## 禁止表达

以下词句会被文本安全工具扫描并替换或触发 WARN：

```text
一定上涨
稳赚
无风险
保本
收益承诺
强烈买入
强烈推荐
确定性机会
```

此外，报告应避免直接使用：

```text
建议买入
建议卖出
建议建仓
看好
收益稳定
策略有效
AI认为可以买
```

## LLM 边界

AI 只能解释 `dashboard.json`、SQLite 和策略参数中已经存在的 `reason/metrics/rule_hits/risk_notes`，不允许自行补充未验证原因。LLM 不能：

- 新增标的。
- 改写 `buy_signal/sell_signal/status/rank/score`。
- 把关注池说成建仓候选。
- 承诺收益。
- 绕过数据质量或风控规则。
- 把历史诊断包装成收益证明。

投研问答默认使用本地规则证据包回答；只有用户点击 `AI 智能问答` 并确认后，才调用 `configs/model_config.json` 中配置的 chat provider 生成更自然的解释。当前默认 chat provider 为 OpenAI，可在前端 `AI 模型` 面板选择模型、遮挡修改 Key 并验证连通。Provider 不可用或超时时，系统必须降级为本地确定性解释，不能阻塞日报生成。

## 固定免责声明

所有 Excel、PDF、dashboard 和问答解释均应保留：

```text
本报告仅为规则模型生成的投研辅助结果，不构成投资建议或收益承诺。
```
# ETF 历史表现诊断口径

ETF 新增的 `historical_diagnostics` 统计状态事件后 5/10/20 个交易日的收益分布、最大有利/不利波动、正收益比例与假反转次数。该模块不模拟完整仓位、成交和资金曲线，因此必须称为“历史表现诊断”，不能包装成可执行回测或保证收益。
