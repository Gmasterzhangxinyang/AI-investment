# 可转债动态策略 v2 设计

## 目标

在保留原可转债质量筛选和硬风控的基础上，引入客户新增的日频联动数据，让排名同时反映基础质量和短期变化，不再只是单日截面比较。

系统提供两个可插拔策略：

- `dynamic_v2`：默认策略，基础质量分与短期动态分结合。
- `legacy_v1`：原策略，保留原评分、资格和排序逻辑，作为对照与回退方案。

策略选择写入配置，不把规则硬编码到页面。切换策略后，后端、页面和问答统一使用同一个当前策略标识。

## 设计原则

1. 原有硬排除、信用风险、赎回风险和资格判断继续生效。
2. 单日动态不能把原本不合格的转债抬入合格候选池。
3. 动态数据只调整合格池内部的优先顺序。
4. 动态字段缺失或校验失败时，不补零、不猜测，自动退回基础质量分。
5. 页面拆开显示基础分、动态分和综合分，避免用户误解分数来源。
6. 新增逻辑独立成策略插件，后续可以只修改插件配置或实现，不影响原策略。

## 数据输入与校验

动态策略读取四个新增字段：

- `stock_daily_return`：正股当日涨幅。
- `bond_daily_return`：转债当日涨幅。
- `previous_conversion_premium_rate`：前日转股溢价率。
- `conversion_premium_change`：转股溢价率当日变化。

系统校验：

`conversion_premium_change ≈ conversion_premium_rate - previous_conversion_premium_rate`

允许误差由配置控制，默认 `0.05` 个百分点。超过误差时标记为“数据待核验”，该标的不计算动态分。

## 两层评分结构

### 基础质量分

基础质量分沿用原策略的长期性价比、基本面、估值、信用、赎回、期限和流动性评价，统一归一化为 `0–100` 分。

基础质量分继续决定：

- 是否通过硬风控；
- 是否属于合格候选；
- 风险等级和原始动作建议。

### 短期动态分

短期动态分为 `0–100` 分，中性值为 `50`，由四类信息组成：

- 正股当日强弱：20%。
- 转债当日强弱：15%。
- 转债相对正股的跟随或背离：30%。
- 溢价率收缩或扩张：35%。

动态分重点评价变化方向，不重复奖励当前溢价率绝对水平。阈值和分段全部放入 `configs/strategy_params.json`，便于后续根据历史样本调整。

主要含义：

- 正股走强、转债跟涨不足、溢价率收缩：偏向“关注补涨”。
- 转债明显跑赢正股且溢价率扩大：偏向“谨慎追涨”。
- 正股和转债同步走弱：偏向“联动走弱”。
- 其余情况：正常联动或中性观察。

### 综合分

动态数据完整且有效时：

`综合分 = 基础质量分 × 80% + 短期动态分 × 20%`

动态数据不足、待核验或插件计算失败时：

`综合分 = 基础质量分`

综合分只用于同一资格池内部排序和 Top10 顺序。资格、硬排除和风险动作仍由基础层决定。

## 插件接口

两个策略实现统一接口：

- 输入：标准化后的可转债数据、策略配置和数据日期。
- 输出：基础分、动态分、综合分、资格、风险状态、动态状态、解释字段和排序字段。

`legacy_v1` 输出原策略分数，动态分为空，综合分等于原策略分数。

`dynamic_v2` 复用公共硬风控和基础质量计算，再调用独立动态评分器生成综合分。动态评分器不直接修改资格和动作。

默认配置：

```json
{
  "convertible_bond": {
    "active_strategy": "dynamic_v2"
  }
}
```

若配置值无效，系统记录明确的数据质量提示，并安全回退到 `legacy_v1`，不能静默输出不确定结果。

## 页面与问答

可转债页面增加策略选择和当前策略说明。

使用 `dynamic_v2` 时显示：

- 基础分；
- 动态分；
- 综合分；
- 动态状态；
- 简短联动提示。

使用 `legacy_v1` 时隐藏动态分和动态状态，分数与排名保持原策略含义。

问答必须说明当前使用的策略，并基于确定性字段解释分数来源。AI 不能自行改变分数、资格和排名。

## 历史数据沉淀

每次导入时保存四个动态原始字段、基础分、动态分、综合分、动态状态和策略版本。当前 v2 只使用当日与前日信息，不宣称已经完成完整历史回测。

当积累足够连续快照后，可以增加 5、10、20 日诊断，检验动态分的正收益比例、平均收益、最大不利波动和排名稳定性。历史诊断作为后续版本，不阻塞 v2 上线。

## 异常与降级

- 旧版 Excel：自动使用基础分，页面提示动态数据不足。
- 部分动态字段缺失：不计算动态分，不按中性值强行混合。
- 溢价率变化校验失败：标记数据待核验，综合分退回基础分。
- 单只标的计算异常：只降级该标的，不影响整批结果。
- 动态插件整体异常：整批安全回退 `legacy_v1`，并显示回退原因。

## 测试与验收

1. 默认配置实际加载 `dynamic_v2`，并可切换到 `legacy_v1`。
2. `legacy_v1` 的分数、资格、排名、动作和 Top10 与当前原策略逐项一致。
3. `dynamic_v2` 的基础分与原策略对应分数一致。
4. 动态数据完整时，综合分严格按 `80% + 20%` 计算。
5. 动态数据缺失或异常时，综合分等于基础分。
6. 动态分不能改变资格池、硬排除和风险动作，只能改变同资格池内排序。
7. 页面会根据当前策略正确显示或隐藏动态列。
8. 问答能说明当前策略、三类分数和降级原因。
9. 全量后端测试、前端测试、语法检查和真实工作流审计通过。

## 不在本期范围

- 不根据单日新字段重写信用和赎回风控。
- 不承诺绝对收益或次日必涨。
- 不把文件名日期替代 Excel 内部数据日期。
- 不在历史样本不足时伪造 5、10、20 日回测结论。
