# TL的30年国债ETF资金流向辅助设计

## 目标

在现有TL日线/周线MACD与KDJ状态诊断旁边，增加30年国债ETF（511090.SH）份额变化的资金流向辅助信息。新信息只用于解释资金是否配合、是否与技术状态背离，不改变现有TL的`entry_candidate`、`attention`、`no_trade`或`neutral`状态。

本次只处理TL。期货持仓量继续保留原始展示，不加入判断；可转债新增字段另行开发。

## 数据输入与兼容

- 使用TL Wind表中的`份额变化（亿份）`列。
- 新文件包含该列时计算资金辅助；旧文件没有该列时，TL原策略照常运行，资金辅助显示“数据不足”。
- 空白单元格按缺失处理；数值`0`按真实零变化处理。若Wind公式把未更新数据自动写成0，数据源维护方需要修正公式，系统同时对连续多日为0给出“可能未更新”提示。
- 原始值完整保留，所有派生判断必须带数据质量状态。
- 稳定配置路径继续使用`02_TL日频公式.xlsx`，不把带日期的快照文件名写死；真实验证时可显式传入`02_TL日频公式-0710快照.xlsx`。

## 日评规则

阈值写入`configs/strategy_params.json`的`tl.fund_flow`配置，默认值为：

- `abs(change) < 0.03`：正常波动。
- `0.03 <= abs(change) < 0.05`：轻度关注。
- `0.05 <= abs(change) < 0.07`：明显申购或明显赎回。
- `0.07 <= abs(change) <= 0.20`：极端申购或极端赎回。
- `abs(change) > 0.20`：统计极端值，标记“待核验”，不计入滚动资金方向，但不删除原始值。

正数表示申购，负数表示赎回。阈值均可配置，暂不根据现有全样本自动寻优。

## 近5日资金趋势

- 使用最近5个有效交易日滚动计算，不使用自然周。
- 待核验极端值和缺失值不进入累计。
- 5日累计大于等于`+0.08亿份`：持续流入。
- 5日累计小于等于`-0.08亿份`：持续流出。
- 其余：资金方向不明显。
- 同时输出5日有效样本数、申购天数和赎回天数。有效样本不足3日时，5日方向标记为“数据不足”。

## 与TL技术状态的关系

资金关系只生成辅助文案：

- TL为建仓候选或关注交易，且资金持续流入：技术与资金同向改善。
- TL为建仓候选或关注交易，但资金持续流出：技术转强与资金流出背离，关注假转强或拐点。
- TL为不做交易，且资金持续流出：技术与资金共同走弱。
- TL为不做交易，但资金持续流入：资金逆势流入，观察潜在转向，但不升级为买点。
- TL为中性或资金方向不明显：只陈述当前资金状态，不作方向推断。

单日明显或极端申赎作为当日补充提示；5日方向作为主要的持续性判断。

## 输出字段

TL今日状态与近期状态新增：

- `fund_share_change_daily`：当日原始份额变化。
- `fund_share_daily_level`：正常、轻度、明显、极端或待核验。
- `fund_share_5d_sum`：近5个有效交易日累计。
- `fund_share_5d_valid_days`：累计采用的有效天数。
- `fund_share_5d_inflow_days`和`fund_share_5d_outflow_days`。
- `fund_flow_state`：持续流入、持续流出、方向不明显或数据不足。
- `fund_flow_relation`：同向改善、技术资金背离、共同走弱、逆势流入或中性。
- `fund_flow_note`：面向用户的中文解释。
- `fund_flow_data_quality`：OK、WARN或UNAVAILABLE。

以上字段通过统一TL状态表进入dashboard、Excel日报、SQLite、AI问答和TL页面，避免多处重复计算。

## 页面展示

TL今日状态卡增加：当日份额变化、当日资金等级、近5日累计、资金关系和提示。

TL近期状态表增加精简的“当日份额变化、5日累计、资金关系”三列。长文案只放在今日状态卡和详情解释，避免表格拥挤。

## 历史诊断与边界

首版仅统计该辅助字段在历史上的状态分布和与后续TL收益的描述性关系，不把它宣传为回测收益，也不自动改变TL规则。只有经过时间对齐、异常值核验、去除未来数据污染和样本外验证后，才讨论是否升级为交易条件。

份额数据只能使用当时已可获得的值。若确认Wind字段为T+1公布，系统必须按可获得日期整体后移一日，不能回填到原交易日用于当天判断。

## 测试与验收

1. 旧TL文件无新增列时，原状态、原因和信号完全不变。
2. 新TL文件能读取`份额变化（亿份）`并生成全部资金辅助字段。
3. `0.03`、`0.05`、`0.07`和`0.20`边界分类正确。
4. 超过`0.20`的原始值保留、标记待核验且不进入5日累计。
5. 缺失值与数值0严格区分；连续零值能产生数据新鲜度警告。
6. 5日累计只使用有效交易日和有效份额值，有效样本不足时不强行判断。
7. 任意资金状态都不改变原TL的`status`、`buy_signal`、`attention_signal`和`no_trade_signal`。
8. 页面、Excel、数据库和AI问答读取同一组结构化字段。
9. 全量自动化测试、新TL真实数据刷新、独立审计和页面视觉检查全部通过。
